# Business Rules - Admin Frontend

## 1. 인증/세션 규칙

### BR-FE-AUTH-01: JWT 토큰 관리
- JWT 토큰은 localStorage에 저장
- 모든 API 요청에 Authorization: Bearer {token} 헤더 포함
- 토큰 만료(16시간) 시 자동 로그아웃
- 401 응답 수신 시 즉시 로그아웃 처리 (토큰 삭제 → 로그인 화면)

### BR-FE-AUTH-02: 로그인 폼 검증
- 매장 식별자: 필수 (UUID 형식)
- 사용자명: 필수, 1~50자
- 비밀번호: 필수, 최소 8자
- 모든 필드 입력 전 로그인 버튼 비활성화

### BR-FE-AUTH-03: 매장 선택 분기
- role=owner → 로그인 후 매장 선택 화면(/stores)으로 이동
- role=staff → 로그인 시 사용한 store_id로 대시보드 바로 진입
- 소유자는 헤더 드롭다운으로 매장 전환 가능
- 직원은 매장 전환 불가

## 2. 역할 기반 접근 제어 (RoleGuard)

### BR-FE-ROLE-01: 역할별 접근 권한
| 기능/페이지 | owner | staff |
|---|---|---|
| 대시보드 (주문 모니터링) | ✅ | ✅ |
| 테이블 관리 (설정, 세션 종료) | ✅ | ✅ |
| 주문 상태 변경 | ✅ | ✅ |
| 주문 삭제 | ✅ | ✅ |
| 메뉴 관리 (CRUD) | ✅ | ❌ |
| 카테고리 관리 (CRUD) | ✅ | ❌ |
| 매장 관리 | ✅ | ❌ |
| 직원 관리 | ✅ | ❌ |
| 매장 전환 | ✅ | ❌ |

### BR-FE-ROLE-02: 접근 제어 구현
- RoleGuard 컴포넌트로 라우트 보호
- 권한 없는 라우트 접근 시 /dashboard로 리다이렉트
- 사이드바 네비게이션에서 권한 없는 메뉴 항목 숨김
- API 호출 전 클라이언트 측 역할 확인 (서버 측 검증은 Backend에서 수행)

## 3. 대시보드 규칙

### BR-FE-DASH-01: 테이블 카드 표시
- 활성 주문이 있는 테이블만 카드로 표시 (동적)
- 주문이 모두 삭제되거나 이용 완료 시 카드 제거
- 카드 정렬: 테이블 번호 오름차순

### BR-FE-DASH-02: 신규 주문 알림
- SSE new_order 이벤트 수신 시:
  - 해당 테이블 카드 하이라이트 (3초간 강조 색상)
  - 사운드 알림 재생 (토글 가능)
- 하이라이트는 3초 후 자동 해제

### BR-FE-DASH-03: SSE 연결 관리
- 대시보드 마운트 시 SSE 연결 시작
- 연결 끊김 시 3초 간격 자동 재연결 (최대 5회)
- 5회 실패 시 "연결 끊김" 배너 표시 + 수동 재연결 버튼
- 대시보드 언마운트 시 SSE 연결 종료
- 매장 전환 시 기존 SSE 종료 → 새 매장 SSE 연결

## 4. 주문 관리 규칙

### BR-FE-ORDER-01: 주문 상태 변경
- 상태 전이 UI: pending → preparing → completed (순방향 버튼)
- 역방향 전이도 허용 (completed → preparing, preparing → pending)
- 상태 변경 시 `PATCH /admin/orders/{order_id}/status` 호출
- 성공 시 UI 즉시 반영, 실패 시 에러 토스트

### BR-FE-ORDER-02: 주문 삭제
- 삭제 버튼 클릭 → 확인 팝업 필수
- 확인 팝업 텍스트: "주문 #{order_number}을(를) 삭제하시겠습니까?"
- 확인 시 `DELETE /admin/orders/{order_id}` 호출
- 성공 시 목록에서 제거 + 총 주문액 재계산
- 실패 시 에러 토스트

### BR-FE-ORDER-03: 주문 금액 표시
- 금액은 원(₩) 단위, 천 단위 콤마 포맷 (예: ₩12,000)
- 총 주문액 = 해당 테이블 활성 세션의 모든 주문 total_amount 합산

## 5. 테이블 관리 규칙

### BR-FE-TABLE-01: 테이블 초기 설정 검증
- 테이블 번호: 필수, 양의 정수, 매장 내 고유
- 비밀번호: 필수, 최소 4자
- 중복 테이블 번호 시 에러 메시지 표시

### BR-FE-TABLE-02: 이용 완료 처리
- 이용 완료 버튼 → 확인 팝업 필수
- 확인 팝업 텍스트: "테이블 #{table_number} 이용을 완료하시겠습니까? 현재 주문이 과거 이력으로 이동됩니다."
- 성공 시 대시보드에서 해당 테이블 카드 제거

### BR-FE-TABLE-03: 과거 내역 조회
- 기본: 오늘 날짜 기준 조회
- 날짜 필터: date_from, date_to (date picker)
- 시간 역순 정렬
- items_snapshot JSON을 파싱하여 메뉴 목록 표시

## 6. 메뉴 관리 규칙 (소유자 전용)

### BR-FE-MENU-01: 메뉴 폼 검증
- 메뉴명: 필수, 1~100자
- 가격: 필수, 0 이상 정수
- 카테고리: 필수, 유효한 category_id 선택
- 설명: 선택, 최대 500자
- 이미지: 선택, JPEG/PNG/WebP, 최대 5MB

### BR-FE-MENU-02: 카테고리 관리 검증
- 카테고리명: 필수, 1~50자
- 카테고리 삭제 시: 하위 메뉴가 있으면 경고 메시지 표시

### BR-FE-MENU-03: 메뉴 순서 조정
- 드래그 앤 드롭 또는 위/아래 버튼으로 순서 변경
- 순서 변경 시 `PATCH /admin/menus/reorder` 호출
- 변경 사항 즉시 반영

### BR-FE-MENU-04: 이미지 업로드
- 클라이언트 측 사전 검증: 파일 타입(JPEG, PNG, WebP), 크기(5MB 이하)
- 검증 실패 시 업로드 차단 + 에러 메시지
- 업로드 중 프로그레스 표시
- 업로드 성공 시 미리보기 표시

## 7. 매장 관리 규칙 (소유자 전용)

### BR-FE-STORE-01: 매장 폼 검증
- 매장명: 필수, 1~100자
- 주소: 선택, 최대 255자
- 연락처: 선택, 최대 20자

### BR-FE-STORE-02: 매장 삭제
- 삭제 버튼 → 확인 팝업 필수
- 확인 팝업 텍스트: "매장을 삭제하시겠습니까? 하위 데이터(메뉴, 테이블, 직원)는 보존됩니다."
- 현재 선택된 매장 삭제 시 매장 선택 화면으로 이동

## 8. 직원 관리 규칙 (소유자 전용)

### BR-FE-STAFF-01: 직원 폼 검증
- 사용자명: 필수, 1~50자
- 비밀번호: 필수(생성 시), 최소 8자
- 비밀번호 수정: 선택 (빈 값이면 변경 안 함)

### BR-FE-STAFF-02: 직원 삭제
- 삭제 버튼 → 확인 팝업 필수
- 자기 자신 삭제 불가

## 9. UI/UX 공통 규칙

### BR-FE-UI-01: 반응형 디자인
- 모바일(320px~), 태블릿(768px~), 데스크톱(1024px~) 브레이크포인트
- 대시보드 그리드: 모바일 1열, 태블릿 2열, 데스크톱 3~4열
- 사이드바: 데스크톱 고정, 모바일/태블릿 햄버거 메뉴

### BR-FE-UI-02: 에러 처리
- API 에러 → 토스트 알림 (3초 자동 닫힘)
- 네트워크 에러 → "네트워크 연결을 확인해주세요" 메시지
- 폼 검증 에러 → 인라인 에러 메시지 (필드 하단)

### BR-FE-UI-03: 로딩 상태
- API 호출 중 로딩 스피너 표시
- 버튼 클릭 후 API 응답 전까지 버튼 비활성화 (중복 요청 방지)

### BR-FE-UI-04: 확인 팝업
- 삭제/이용 완료 등 파괴적 작업 시 확인 팝업 필수
- 팝업에 "취소" + "확인" 버튼
- 확인 버튼은 위험 색상(빨간색 계열)으로 표시
