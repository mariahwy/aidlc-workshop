## Clarifying Questions

### Q1: 대시보드 레이아웃 및 테이블 카드 표시 방식
대시보드에서 테이블 카드의 기본 정렬 및 표시 방식을 어떻게 할까요?

- A) 테이블 번호 순서대로 고정 그리드 (빈 테이블도 표시)
- B) 활성 주문이 있는 테이블만 표시 (동적)
- C) 모든 테이블 표시하되, 활성 주문이 있는 테이블을 상단에 강조 표시

[Answer]: B

### Q2: 테이블 상세 모달 vs 페이지 전환
테이블 카드 클릭 시 주문 상세를 어떻게 표시할까요?

- A) 모달/오버레이로 표시 (대시보드 위에 겹침)
- B) 별도 페이지로 이동
- C) 사이드 패널로 표시 (대시보드 옆에 슬라이드)

[Answer]: A

### Q3: 메뉴 관리 - 카테고리 CRUD
메뉴 관리에서 카테고리 자체의 생성/수정/삭제 기능도 포함할까요?

- A) 카테고리 CRUD 포함 (카테고리 생성, 이름 수정, 삭제, 순서 조정)
- B) 카테고리는 사전 정의된 것만 사용 (메뉴 항목 CRUD만)
- C) 카테고리 생성/삭제만 (수정 없음)

[Answer]: A

### Q4: 매장 선택 방식
다중 매장 지원 시, 로그인 후 매장 선택 흐름을 어떻게 할까요?

- A) 로그인 시 매장 ID를 입력하므로 해당 매장으로 바로 진입
- B) 로그인 후 매장 목록에서 선택하는 별도 화면 제공
- C) 소유자는 매장 전환 가능, 직원은 소속 매장 자동 진입

[Answer]: C

### Q5: 알림/사운드
신규 주문 도착 시 시각적 강조 외에 사운드 알림도 필요할까요?

- A) 시각적 강조만 (색상/애니메이션)
- B) 시각적 강조 + 사운드 알림
- C) 시각적 강조 + 사운드 알림 + 브라우저 알림(Notification API)

[Answer]: B

### Q6: 반응형 디자인
관리자 앱의 주요 사용 디바이스는 무엇인가요?

- A) 데스크톱/노트북 전용 (최소 1024px)
- B) 태블릿 + 데스크톱 반응형 (최소 768px)
- C) 모바일 포함 완전 반응형

[Answer]: C

### Q7: 다국어 지원
관리자 앱의 언어 지원 범위는?

- A) 한국어 전용
- B) 한국어 + 영어 (i18n 구조 포함)
- C) 다국어 확장 가능한 i18n 구조만 준비 (초기 한국어만)

[Answer]: A
