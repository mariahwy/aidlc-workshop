# 사용자 페르소나

## 페르소나 1: 고객 (Customer)

| 항목 | 내용 |
|---|---|
| **이름** | 김민수 (대표 고객) |
| **역할** | 매장 방문 고객 (테이블 이용자) |
| **설명** | 매장에 방문하여 테이블에 앉아 태블릿으로 음식을 주문하는 일반 고객 |
| **목표** | 빠르고 편리하게 메뉴를 탐색하고 주문하기 |
| **동기** | 직원 호출 없이 즉시 주문, 대기 시간 최소화 |
| **불만사항** | 복잡한 UI, 느린 응답, 주문 상태 확인 불가, 장바구니 유실 |
| **기술 수준** | 낮음~중간 (스마트폰 사용 가능 수준) |
| **사용 환경** | 매장 내 테이블 태블릿 (터치 기반 웹 브라우저) |

### 핵심 니즈
- 별도 로그인 없이 즉시 주문 가능
- 카테고리별 메뉴 탐색 및 이미지 확인
- 장바구니에서 수량 조절 후 한 번에 주문
- 주문 후 상태(대기중/준비중/완료) 실시간 확인
- 페이지 새로고침 시에도 장바구니 유지

## 페르소나 2: 매장 소유자 (Store Owner)

| 항목 | 내용 |
|---|---|
| **이름** | 박지영 (대표 매장 소유자) |
| **역할** | 매장 소유자 / 최고 관리자 |
| **설명** | 매장을 소유하고 전체 운영을 관리하는 사업주 |
| **목표** | 매장 운영 효율화, 메뉴/테이블/직원 전체 관리 |
| **동기** | 인건비 절감, 운영 데이터 기반 의사결정, 다중 매장 관리 |
| **불만사항** | 주문 누락, 비효율적 테이블 관리, 과거 데이터 조회 어려움 |
| **기술 수준** | 중간 (웹 기반 관리 도구 사용 가능) |
| **사용 환경** | 데스크탑/노트북 웹 브라우저 |

### 핵심 니즈
- 매장 등록 및 관리
- 메뉴 CRUD (등록/수정/삭제/순서 조정)
- 테이블 초기 설정 및 세션 관리
- 실시간 주문 모니터링
- 과거 주문 내역 조회 및 분석
- 매장 직원 계정 관리

### 권한
- 모든 관리 기능 접근 가능
- 매장 등록/수정/삭제
- 직원 계정 생성/관리
- 메뉴/테이블/주문 전체 관리

## 페르소나 3: 매장 직원 (Store Staff)

| 항목 | 내용 |
|---|---|
| **이름** | 이준호 (대표 매장 직원) |
| **역할** | 매장 직원 / 운영 담당자 |
| **설명** | 매장에서 일하며 주문 접수 및 테이블 관리를 담당하는 직원 |
| **목표** | 실시간 주문 확인 및 빠른 처리, 테이블 회전율 관리 |
| **동기** | 효율적 업무 처리, 주문 누락 방지, 고객 만족 |
| **불만사항** | 주문 알림 지연, 테이블 상태 파악 어려움, 복잡한 관리 화면 |
| **기술 수준** | 낮음~중간 |
| **사용 환경** | 매장 내 데스크탑/태블릿 웹 브라우저 |

### 핵심 니즈
- 실시간 주문 모니터링 및 상태 변경
- 테이블 세션 관리 (이용 완료 처리)
- 주문 삭제 (직권 수정)
- 과거 주문 내역 조회

### 권한
- 주문 모니터링 및 상태 변경
- 테이블 세션 관리
- 주문 삭제
- 메뉴 조회 (수정 불가)
- 매장 설정 변경 불가
