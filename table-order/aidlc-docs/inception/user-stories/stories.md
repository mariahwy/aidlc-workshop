# 사용자 스토리

## 고객용 스토리 (Customer Stories)

### US-C01: 테이블 자동 로그인
**우선순위**: High
**페르소나**: 고객

> 고객으로서, 테이블에 앉았을 때 별도 로그인 없이 즉시 주문 화면을 볼 수 있기를 원한다. 그래야 대기 시간 없이 바로 주문할 수 있다.

**수용 기준**:
- Given 관리자가 테이블 초기 설정을 완료한 상태일 때, When 태블릿 브라우저를 열면, Then 저장된 인증 정보로 자동 로그인되어 메뉴 화면이 표시된다
- Given 자동 로그인에 실패한 경우, When 태블릿 브라우저를 열면, Then 매장 식별자/테이블 번호/비밀번호 입력 화면이 표시된다
- 로그인 성공 시 인증 정보가 로컬에 저장된다
- 16시간 세션이 유지된다

---

### US-C02: 메뉴 조회 및 탐색
**우선순위**: High
**페르소나**: 고객

> 고객으로서, 카테고리별로 메뉴를 탐색하고 상세 정보를 확인할 수 있기를 원한다. 그래야 원하는 메뉴를 쉽게 찾아 선택할 수 있다.

**수용 기준**:
- Given 로그인된 상태일 때, When 메뉴 화면에 접근하면, Then 카테고리별로 분류된 메뉴 목록이 카드 형태로 표시된다
- Given 메뉴 목록이 표시된 상태일 때, When 카테고리를 선택하면, Then 해당 카테고리의 메뉴만 필터링되어 표시된다
- 각 메뉴 카드에 메뉴명, 가격, 설명, 이미지가 표시된다
- 버튼 크기가 최소 44x44px 이상이다
- 메뉴 화면이 기본(홈) 화면이다

---

### US-C03: 장바구니 관리
**우선순위**: High
**페르소나**: 고객

> 고객으로서, 주문 전에 선택한 메뉴를 장바구니에서 확인하고 수정할 수 있기를 원한다. 그래야 실수 없이 정확한 주문을 할 수 있다.

**수용 기준**:
- Given 메뉴 화면에서, When 메뉴 항목을 장바구니에 추가하면, Then 장바구니에 해당 메뉴가 추가되고 총 금액이 업데이트된다
- Given 장바구니에 메뉴가 있을 때, When 수량을 변경하면, Then 수량과 총 금액이 실시간으로 재계산된다
- Given 장바구니에 메뉴가 있을 때, When 삭제 버튼을 누르면, Then 해당 메뉴가 장바구니에서 제거된다
- 장바구니 비우기 기능이 제공된다
- 페이지 새로고침 시에도 장바구니 내용이 유지된다 (로컬 저장)
- 장바구니 데이터는 주문 확정 시에만 서버로 전송된다

---

### US-C04: 주문 생성
**우선순위**: High
**페르소나**: 고객

> 고객으로서, 장바구니의 메뉴를 확인하고 주문을 확정할 수 있기를 원한다. 그래야 원하는 음식을 주문할 수 있다.

**수용 기준**:
- Given 장바구니에 메뉴가 있을 때, When 주문 확정 버튼을 누르면, Then 주문 내역 최종 확인 화면이 표시된다
- Given 주문 확인 화면에서, When 최종 확정하면, Then 주문이 서버로 전송된다
- Given 주문이 성공하면, Then 주문 번호가 표시되고, 장바구니가 비워지고, 5초 후 메뉴 화면으로 자동 리다이렉트된다
- Given 주문이 실패하면, Then 에러 메시지가 표시되고 장바구니는 유지된다
- 주문 정보에 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID가 포함된다

---

### US-C05: 주문 내역 조회
**우선순위**: Medium
**페르소나**: 고객

> 고객으로서, 현재 테이블에서 주문한 내역과 상태를 확인할 수 있기를 원한다. 그래야 주문 진행 상황을 알 수 있다.

**수용 기준**:
- Given 로그인된 상태일 때, When 주문 내역 화면에 접근하면, Then 현재 테이블 세션의 주문 목록이 시간 순으로 표시된다
- 각 주문에 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)가 표시된다
- 이전 세션(이용 완료 처리된) 주문은 표시되지 않는다
- 매장 이용 완료 처리된 주문은 제외된다

---

## 관리자용 스토리 (Admin Stories)

### US-A01: 매장 관리자 인증
**우선순위**: High
**페르소나**: 매장 소유자, 매장 직원

> 매장 관리자로서, 매장 식별자와 계정 정보로 로그인할 수 있기를 원한다. 그래야 매장 관리 시스템에 안전하게 접근할 수 있다.

**수용 기준**:
- Given 관리자 로그인 화면에서, When 매장 식별자/사용자명/비밀번호를 입력하고 로그인하면, Then JWT 토큰이 발급되고 대시보드로 이동한다
- Given 잘못된 인증 정보를 입력하면, Then 에러 메시지가 표시된다
- 16시간 세션 유지 (JWT 토큰)
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호는 bcrypt로 해싱 저장
- 로그인 시도 제한 (brute-force 방지)

---

### US-A02: 실시간 주문 모니터링
**우선순위**: High
**페르소나**: 매장 소유자, 매장 직원

> 매장 관리자로서, 들어오는 주문을 실시간으로 확인하고 상태를 변경할 수 있기를 원한다. 그래야 주문을 빠르게 처리할 수 있다.

**수용 기준**:
- Given 대시보드 화면에서, When 새 주문이 들어오면, Then 2초 이내에 해당 테이블 카드에 주문이 표시된다 (SSE)
- Given 테이블별 카드가 표시된 상태에서, When 카드를 클릭하면, Then 해당 테이블의 전체 주문 메뉴 목록이 상세 표시된다
- 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
- 주문 상태를 대기중→준비중→완료로 변경할 수 있다
- 신규 주문은 시각적으로 강조 표시된다 (색상/애니메이션)
- 그리드 형태의 테이블별 레이아웃이다

---

### US-A03: 테이블 관리
**우선순위**: High
**페르소나**: 매장 소유자, 매장 직원

> 매장 관리자로서, 테이블 설정과 세션을 관리할 수 있기를 원한다. 그래야 고객 회전을 효율적으로 관리할 수 있다.

**수용 기준**:

**테이블 초기 설정:**
- Given 테이블 관리 화면에서, When 테이블 번호와 비밀번호를 설정하면, Then 16시간 세션이 생성되고 자동 로그인이 활성화된다

**주문 삭제:**
- Given 테이블 주문 목록에서, When 주문 삭제 버튼을 누르면, Then 확인 팝업이 표시된다
- Given 확인 팝업에서 확인을 누르면, Then 주문이 즉시 삭제되고 총 주문액이 재계산된다

**테이블 이용 완료:**
- Given 테이블 카드에서, When 이용 완료 버튼을 누르면, Then 확인 팝업이 표시된다
- Given 확인 팝업에서 확인을 누르면, Then 현재 세션 주문이 과거 이력으로 이동하고, 현재 주문 목록과 총 주문액이 0으로 리셋된다

**과거 내역 조회:**
- Given 테이블 카드에서, When 과거 내역 버튼을 누르면, Then 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다
- 날짜 필터링 기능이 제공된다
- 닫기 버튼으로 대시보드 복귀가 가능하다

---

### US-A04: 메뉴 관리
**우선순위**: High
**페르소나**: 매장 소유자

> 매장 소유자로서, 메뉴를 등록/수정/삭제하고 노출 순서를 조정할 수 있기를 원한다. 그래야 매장 메뉴를 유연하게 관리할 수 있다.

**수용 기준**:
- Given 메뉴 관리 화면에서, When 카테고리를 선택하면, Then 해당 카테고리의 메뉴 목록이 표시된다
- Given 메뉴 등록 화면에서, When 메뉴명/가격/설명/카테고리/이미지를 입력하고 저장하면, Then 새 메뉴가 등록된다
- 메뉴 수정 및 삭제가 가능하다
- 메뉴 노출 순서를 드래그 또는 버튼으로 조정할 수 있다
- 필수 필드(메뉴명, 가격, 카테고리) 검증이 수행된다
- 가격 범위 검증이 수행된다 (0 이상)
- 메뉴 이미지는 S3에 업로드되고 URL이 저장된다

---

### US-A05: 매장 관리
**우선순위**: High
**페르소나**: 매장 소유자

> 매장 소유자로서, 매장을 등록하고 관리할 수 있기를 원한다. 그래야 다중 매장을 효율적으로 운영할 수 있다.

**수용 기준**:
- Given 매장 관리 화면에서, When 새 매장 정보를 입력하고 등록하면, Then 새 매장이 생성된다
- 매장 정보 수정이 가능하다
- 매장 삭제가 가능하다 (확인 팝업 포함)
- 각 매장은 독립적인 메뉴, 테이블, 주문 데이터를 가진다
- 매장 목록에서 매장을 선택하여 해당 매장의 관리 화면으로 진입할 수 있다

---

### US-A06: 매장 직원 계정 관리
**우선순위**: Medium
**페르소나**: 매장 소유자

> 매장 소유자로서, 매장 직원 계정을 생성하고 관리할 수 있기를 원한다. 그래야 직원에게 적절한 권한을 부여할 수 있다.

**수용 기준**:
- Given 직원 관리 화면에서, When 직원 계정 정보를 입력하고 생성하면, Then 직원 역할의 계정이 생성된다
- 직원 계정 수정 및 삭제가 가능하다
- 직원 역할은 주문 모니터링, 테이블 세션 관리, 주문 삭제 권한만 가진다
- 직원은 메뉴 관리, 매장 설정, 직원 관리에 접근할 수 없다

---

## 시스템 스토리 (System Stories)

### US-S01: 메뉴 이미지 S3 업로드
**우선순위**: Medium
**페르소나**: 매장 소유자

> 매장 소유자로서, 메뉴 이미지를 업로드하면 클라우드에 안전하게 저장되기를 원한다. 그래야 고객에게 메뉴 이미지를 안정적으로 제공할 수 있다.

**수용 기준**:
- Given 메뉴 등록/수정 화면에서, When 이미지 파일을 선택하고 업로드하면, Then 이미지가 S3에 저장되고 URL이 반환된다
- 반환된 URL이 메뉴 데이터에 저장된다
- 업로드 실패 시 에러 메시지가 표시된다
